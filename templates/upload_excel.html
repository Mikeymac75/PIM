Instructions for Excel File Format:

Please ensure your Excel file (.xlsx format) has the following columns in the first sheet:

  • Name: (Required) The name of the grocery item (Text).
  • Quantity: (Required) The quantity of the item (Text or Number, e.g., "10 bags", "2", "1.5 liters").
  • Purchase Date: (Required) The date of purchase in YYYY-MM-DD format (e.g., 2023-10-28).
  • Expiry Days: (Required) The number of days from the purchase date until the item expires (Number, e.g., 7, 14, 30).
  • Category: (Required for new products) The category of the item (Text, e.g., "Dairy", "Produce").
  • Subcategory: (Optional) The subcategory of the item (Text, e.g., "Milk", "Fruit").
  • Par Level: (Optional) The minimum desired quantity in stock (Number, e.g., 2, 0.5). Defaults to 0 if blank/missing. Must be non-negative.
  • Max Holding Amount: (Optional) The maximum preferred quantity to stock (Number, e.g., 4, 1). Defaults to 0 if blank/missing. Must be non-negative.
  • Purchase Location: (Optional) The store or source of purchase (Text, e.g., "Sobeys", "Costco"). If provided, must be one of the allowed locations.
  • Unit of Measure: (Conditionally Required) The unit of measure for the item (Text, e.g., "kg", "lbs", "pcs", "liters"). This is required if the item is being added for the first time (i.e., does not exist as a product yet). If the product already exists, this column is optional and will be ignored.
  • Cost_Per_Unit: (Optional) The cost for a single unit of the item (Number, e.g., 2.99). If provided, the purchase will be logged with cost information. Must be non-negative.
  • Vendor: (Optional) The name of the store or vendor where the item was purchased (Text). Often the same as Purchase Location but can be more specific.

The first row of the sheet should be the header row. The order of columns does not strictly matter as long as the names are recognized (case-insensitive, use underscores for multi-word headers like cost_per_unit).
